# Учим грузинский алфавит

Интерактивное веб-приложение для запоминания букв грузинского алфавита Мхедрули. Проект помогает сопоставлять грузинские символы с русскими звуковыми подсказками и содержит отдельный справочник всех букв с мнемониками.

## Возможности

- Тренировочная сессия из 10 вопросов.
- Случайный выбор грузинской буквы и четырех вариантов ответа.
- Варианты ответа отображаются русскими строчными буквами.
- Мгновенная обратная связь после ответа.
- Итоговый экран с количеством правильных ответов и историей прохождения.
- Экран `Все буквы` со списком 33 символов, транскрипцией и русскими мнемониками.
- Адаптивный интерфейс для мобильных и десктопных экранов.

## Технологии

- [Vue 3](https://vuejs.org/) через CDN.
- [Pinia 3](https://pinia.vuejs.org/) через CDN.
- [Tailwind CSS](https://tailwindcss.com/) через CDN.
- Google Fonts: `Noto Sans Georgian`.

Проект не требует сборки, пакетного менеджера или установки зависимостей. Все внешние библиотеки подключаются в `index.html`.

## Быстрый старт

Откройте `index.html` в браузере.

Для запуска через локальный HTTP-сервер можно использовать любой статический сервер, например:

```bash
python3 -m http.server 8000
```

После запуска откройте в браузере:

```text
http://localhost:8000
```

## Требования

- Современный браузер с поддержкой JavaScript.
- Доступ к интернету для загрузки Vue, Pinia, Tailwind CSS и шрифта из CDN.

## Как устроено приложение

Основные данные находятся в массиве `GEORGIAN_ALPHABET`. Каждая запись содержит:

- `letter` - грузинская буква.
- `trans` - латинская транскрипция.
- `ru` - русское строчное обозначение звука для вариантов ответа.
- `mnemonic` - подсказка для запоминания формы буквы.

Состояние приложения хранится в Pinia store `game`. Store управляет текущим экраном, прогрессом сессии, счетом, вариантами ответа, обратной связью и историей ответов.

## Основные экраны

- `start` - стартовый экран с переходом к тренировке и справочнику.
- `letters` - список всех букв с мнемониками.
- `quiz` - экран вопроса.
- `results` - итоговый экран прохождения.

## Разработка

Поскольку проект реализован как один статический HTML-файл, изменения вносятся напрямую в `index.html`.

Рекомендуемый порядок работы:

1. Обновить данные или логику в `index.html`.
2. Открыть страницу в браузере или обновить текущую вкладку.
3. Проверить сценарии: старт, справочник букв, прохождение теста, экран результатов.
